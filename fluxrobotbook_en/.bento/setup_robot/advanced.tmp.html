automatic program download

  1. Type in – FTP -i 192.168.1.175 and press Enter

  1. If connected you would get the above response. Now press ENTER again to login

  2. Type cd md: to access the robot controller directory

  3. To retrieve a program from controller use mget <program name>.ls

  4. To send a program from laptop to controller use mput <program name>.ls

Preparing RA for the bend:

  1. Create a program with same parameters as used in flux or use the flux program file to load program in RA.

  2. To Run the Press Brake in Robot Mode Refer DOC – Robot Interface Implementation.PDF ( available in shared drive).

  3. To achieve the synchronisation the Ram speed during bend should be set to 50% i.e, 5mm/sec. This can be modified in the EDIT CYCLE option.

Preparing ROBOT for the bend:

  1. To Run the program, currently we are running only in MANUAL Mode. This manual mode has 2 options T1 mode and T2 mode.

  2. T1 mode : Robot speed is set to 250mm/sec as its max limit.

  3. T2 mode : Robot speed is set to 2000mm/sec which is the top speed of this robot.

  4. While testing the robot with synchro Bend, T2 mode should be used to achieve the syncro motion during the bend.

  5. Now open the AROBO program from the program menu in the robot teach pendent.

  6. If the robot is in T2 mode you can observe that speed is automatically set to 3%, Before start change it to 100%. Speed for individual motion is controlled inside the logic by manipulating register R[5] for joint motion R[6] for Linear Motion.

  7. Once the deadman switch is released this toggles the speed back to 3%, only in T2 mode.

  8. Now you can press shift+fwd in teach pendent to start the program.

  9. During fastclosing foot pedel down by operator is required.

  10. If error occurs in robot this will cause the program to abort, so you have to reset the Press brake by pressing up pedal and reset the program in the TP.
