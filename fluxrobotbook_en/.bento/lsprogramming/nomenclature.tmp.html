Program Names:

Main program is always "name saved in flux".ls

Bend"bend number"Positioning_sub.ls - specifies the positioning of the part for corresponding bend for bend no 1 the name will be Bend1Positioning_sub.ls.

Bend"bend num"Sub.ls - specifies the sub program name denotion for bend syncronisation program, for bend no 1 the name would be Bend1Sub.ls

Data Registers are used in the program to manipulate motion speeds and manage program states

R[2:ANA_VAL] - Analog feedback value from Z1 axes potentiomater

R[2:ANA_VAL] - Analog feedback value from Z2 axes potentiomater

R[15:SPD_J] - Speed of Joint type motion point (%)

R[16:SPD_L] - Speed of Linear type motion point (mm/sec)

R[17:BendNumStatus] - State data for bend positioning, specifies the bend number in a multi bend program

R[10:ProgramState] - State data for main program

R[18:GripperType] - Specifies type of gripper used, 0 means Vacuum Gripper and 1 means Pinch gripper

R[33] - For loop variable

R[34:CommandFromPressBrake] - Motion syncro command from Press Brake Controller

R[35:BendPointCommand] - syncro command used in motion line

R[38:counter] - Loop Counter for syncro motion
