A good gripper is often the key to a good bend sequence in FluxBend. In order to find an efficient solution you might take minigripper.STL and modify the coordinates of the suction cups until you have a good solution. Example material is in 04_components\Gripper. The STL file (binary) represents the collision geometry, the dat file the position of the geometrical properties, information about the suction cup position, group and type.

The recommended steps are the following:

  • Try to find a good bending solution by either exiting grippers in your library

  • If there is not fiting gripper do a placement test of the cup positions in a gripper.dat file until you have a good solution. You see the accesibility of the gripper and collisions of the robot with the machine.

  • of if you go for a laser cut gripper you could modify the example drawing Gripper45-h100r160-p1.fx

  • laser cut the parts to assemble a gripper

  • measure the height of the clutch in the activated state (suction on) and enter the value in the gripper.dat file

  • run the Matlab Gripper tool which creates you the collision contour of the gripper

  • import the gripper in FluxBend via Database > Bend Grippers

  • check again your whole bending process, now Fluxrobot considers also the collisions of the gripper geometry
